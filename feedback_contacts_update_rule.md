**매주 Contacts 업데이트 규칙:**

1. **관제 크롤 → 전체 덮어쓰기** — 기사들이 번호를 업데이트하는 경우 있으므로 매주 관제 데이터로 전부 덮어쓰기 (rename + new)
2. **HARDCODE_BY_FILENAME 인원은 예외** — 사용자가 직접 번호 변경을 지시한 기사는 관제 크롤로 번호 변경하지 말 것. HARDCODE에 등록된 번호가 최종 정답.

**HARDCODE 등록 사유:** 기사들 중 연락처를 자주 바꾸는 경우가 많아서 관제 데이터가 최신이 아닐 수 있음. 사용자가 직접 확인한 번호를 HARDCODE로 고정.

**현재 HARDCODE 번호 변경 이력:**
- 인태식6742 → 010-7675-6742
- 진홍희1455 → 010-8146-1455
- 이유신6314 → 010-8072-6314
- 유영진7130 → 010-2564-4376

**Why:** 관제 사이트는 기사 본인이 업데이트하므로 대부분 최신이지만, 일부 기사는 번호를 바꾸고 관제에 반영 안 함. 사용자가 직접 확인한 번호가 가장 정확.
**How to apply:** Contacts 업데이트 diff 시 HARDCODE_BY_FILENAME에 등록된 파일의 번호는 스킵. 나머지는 관제 기준 덮어쓰기.

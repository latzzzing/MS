**기존 자료를 지우는 것은 절대로 안 됨.** 자동화 도구가 어떤 셀에든 *기존 값이 있는데 덮어쓰는* 동작은 절대 금지.

**Why:** 임대 내역 같은 공유 장부는 여러 사람이 보고 편집하는 *살아있는 기록*. 한 번 덮어쓰면 복구 어려움 (OneDrive 버전 히스토리 있어도 즉시 인식 안 되면 바로 알 길 없음). 사용자가 명시적으로 보낸 신호는 "추가만 OK, 수정/삭제 NO".

**How to apply:**
- 신규 기사 등록 같은 *추가* 작업: 빈 컬럼/행에만 쓰기. 마지막 사용 셀 탐지를 보수적으로 (모든 행 스캔, 2-3행만 보는 식 X).
- 어떤 셀에 쓰기 전: 그 셀이 진짜로 비어있는지 (`is None`) 직접 확인. 비어있지 않으면 즉시 중단 + 사용자에게 알림.
- 컬럼/시트 *삭제* 도구는 만들지 말 것. 정말 필요하면 사용자가 수동으로.
- 자동 백업은 항상 (예: `/tmp/<file>_backup_<timestamp>.xlsx`) 도구 실행 직전.
- "수정" 작업 (기존 셀 값 변경) 은 사용자가 *명시적으로* 그 셀의 옛 값과 새 값을 모두 알려줄 때만 가능. 추측으로 덮어쓰기 X.

**적용 대상:**
- 임대 내역.xlsx (`add_new_rider` 등)
- 정산서 폴더의 .xlsx 들
- 향후 추가될 모든 장부/시트 자동화 도구

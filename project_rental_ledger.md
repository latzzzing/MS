**파일:** `/Users/settlment/Library/CloudStorage/OneDrive-개인(2)/공유/[2026년] 임대 내역.xlsx`

**시트 구조:** 월별 시트 (`렌탈 YYMM월` — 예: `렌탈 2605월`). 한 달 = 전월 29일 ~ 당월 26일 (28일 사이클). 시트 내 행 4부터 일자 행 시작 (`29일`, `30일`, `1일`, ...).

**기사 컬럼 배치 (보증금 모드 2종):**
- **완납** (=선납, 보증금 미리 다 받음) → **2열**: `이름` | `미납잔액`
- **일차감** (보증금 받아두고 매일 2만원씩 차감하며 운영) → **3열**: `이름` | `미납잔액` | `보증금`
  - 매일 2만원 차감은 *수식에 박혀있는 고정값* (사용자 입력 X). 사용자는 보증금 *시작액* (보통 10만/20만) 만 지정.
- ❌ "할부" 라는 표현은 안 씀. "일차감" 이 표준 용어.
- 각 기사 단위 사이에 *빈 컬럼 1개* (구분 간격) 두기

**행 구조:**
- 행 2: 이름 / 미납잔액 / (보증금)
- 행 3: 단가 (`일25000`, `주140000` 등) / 시작 잔액 (보통 0) / (보증금 시작액)
- 행 4 ~ 마지막 일자: 매일 입금액(또는 `미`/`면제`) / 자동 수식

**컬럼 폭 (필수):**

xlsx 의 stored width 값과 Excel UI 표시값은 다름. 기존 컬럼 패턴을 그대로 복제하는 것이 안전.

**openpyxl 으로 셋업 시 stored 값:**
- 구분 간격 컬럼 (앞 빈 칸): `2.6640625` (Excel UI 표시 ~1.84)
- 이름 컬럼: `12.1640625`
- 미납잔액 컬럼: `12.1640625`
- 보증금 컬럼 (3열 모드 한정): `12.1640625` (가정 — 첫 3열 모드 등록 시 기존 보증금 컬럼 stored 값 확인 후 갱신)

즉 코드에서:
```python
ws.column_dimensions["<sep>"].width = 2.6640625
ws.column_dimensions["<name>"].width = 12.1640625
ws.column_dimensions["<balance>"].width = 12.1640625
# 3열 모드: ws.column_dimensions["<deposit>"].width = 12.1640625
```

사용자가 "1.84, 12.5, 12.5" 라고 말하는 건 Excel UI 표시값 기준. **절대 stored 에 그대로 박지 말 것** — Excel 이 더 좁게 표시하게 됨 (실측 사례: stored 1.84 → Excel UI 1).

**미납잔액 수식 (2열 완납 모드):**
```
=IF(<일별셀>="면제",<전행 미납잔액>,IF(<일별셀>="미",<전행 미납잔액>+<일렌트>,<전행 미납잔액>+<일렌트>-<일별셀>))
```
- 일렌트는 **숫자로 직접 박음** (RIGHT(rate$3,5) 같은 셀 참조 사용 X). **이유:** 단가 변경 시 옛 기록이 함께 변하면 안 됨. 등록 시점의 단가를 그 자리에 고정.
- 단가 문자열에서 정규식 `\d+` 마지막 매치로 숫자 추출 (예: "일24000" → 24000, "주140000" → 140000).

**미납잔액/보증금 수식 (3열 모드 = 일차감):**

기존 패턴 확인 완료 (예: 권세준 FX/FY/FZ, 김하람 FE/FF/FG, 임찬혁 FP/FQ/FR 등 — 렌탈 2604월 시트).

미납잔액 (둘째 컬럼) 수식:
```
=IF(<일별>="미", <전행 잔액>+<일렌트>, <전행 잔액>-(<일별>-<전행 보증금>+<오늘 보증금>)+<일렌트>)
```
- 일렌트는 **숫자로 직접 박음** (RIGHT 사용 X). 단가 문자열에서 마지막 숫자 (정규식 `\d+` 마지막 매치) 로 추출.
- 예: '일20000' → 20000, '출근시 5.2\n32000책임' → 32000

보증금 (셋째 컬럼) 수식:
```
=IF(<일별>="미", <전행 보증금>, <전행 보증금>-IF(<일별>>=20000, 20000, <일별>))
```
- 미납이면 보증금 그대로, 그 외엔 입금액과 20000 중 작은 값을 보증금에서 차감.
- 20000 이 "기본 일 2만원 차감" 의 정체.

봇 (`_add_new_rider`) 에 위 패턴 양쪽 다 구현 완료 (2026-05-03).

**기사 라이프사이클 룰 (사용자 명시):**
- **계약일** = 렌트료 0원. 일별 입금 셀은 공란 또는 0. 미납잔액 = 0 (수식 없음).
- **반납일** = 하루치 발생. 일별 셀에 `이름 반납` (텍스트), 미납잔액 = (직전 잔액 + 1일 단가) **static 값** (수식이면 텍스트로 인한 #VALUE! 오류).

**신규 계약 등록 절차** (사용자 용어: *"신규 계약"*. "신규 기사"/"기사 등록" 같은 표현은 사용자 피드백으로 사용 금지):
1. 오늘 (또는 명시된 계약일) 기준 영업월 `렌탈 YYMM월` 시트 선택 (영업월 = 전월 29 ~ 당월 26 사이클)
2. 마지막 사용 컬럼 *모든 행 스캔* 으로 결정. 다음에 빈 컬럼 1개 두고 신규 계약 컬럼 배치.
3. **쓰기 직전 빈 셀 검증** — 신규 후보 컬럼들이 모든 행에서 None 인지 확인. 아니면 즉시 중단 (덮어쓰기 방지).
4. 행 2/3 헤더 채움
5. 계약일 행에 `이름 계약` 표기 + 미납잔액 = 0 (3열이면 보증금 = 시작액)
6. **계약일 다음 행 한 칸만** 미납잔액(+ 보증금) 수식 채움. 그 이후 행은 비움 — 여직원이 수동 입력.
7. 컬럼 폭 위 규격대로 설정
8. 자동 백업 (`/tmp/임대내역_backup_<timestamp>.xlsx`)

**여직원 운영:** 매일 일별 셀에 입금액 또는 `미`(미납)/`면제` 입력 → 미납잔액 자동 갱신. 봇은 등록만 담당.
